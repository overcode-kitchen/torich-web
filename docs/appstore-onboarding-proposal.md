# 토리치 앱스토어 이미지 & 온보딩 기획 제안

## 요약
- **앱스토어 스크린샷 5장**: 훅 → 핵심 기능 2개 → 차별화 → CTA 순으로, 기존 화면/컴포넌트 재사용 위주 제안.
- **온보딩**: 현재 랜딩 2섹션만 존재. 앱스토어 메시지와 맞춰 4~5단계로 확장하고, 로그인 후 짧은 온보딩(알림 권한 등) 추가 제안.

---

# Part 1. 앱스토어 스크린샷 5장 제안

## 공통 원칙
- 실제 앱 화면을 캡처하거나, 동일 컴포넌트로 제작해 **실제 사용 경험과 일치**시키기.
- 메인 타이틀 + 서브 텍스트는 이미지 위 오버레이 또는 앱스토어 텍스트 영역에 사용.

---

## 1장: 서비스 핵심 가치 전달 (훅)

**목적:** "매달 투자 까먹지 않게 관리해준다"는 한 문장으로 관심 끌기.

| 항목 | 내용 |
|------|------|
| **활용 페이지/컴포넌트** | `app/components/LandingPageSections/HeroSection.tsx` |
| **추천 캡처 영역** | 히어로 카피 + "토리치가 해주는 일" 흰 카드(투자일 알림, 납입 확인, 쌓인 금액 한눈에)까지. 하단 CTA 버튼은 생략하거나 블러 처리 가능. |
| **메인 타이틀** | 매달 투자, 까먹지 않게 관리해줄게 |
| **서브 텍스트** | 적립식 투자를 캘린더처럼 관리해주는 작은 투자 동반자, 토리치예요. |
| **강조할 UI 요소** | 상단 헤드라인 텍스트, 3가지 벌릿(도토리 아이콘 + 문장). 토리 그래픽(grab/grab_hand)은 분위기용으로 유지. |

**디자인 노트:** 기존 히어로와 동일한 레이아웃 사용. 앱스토어용으로는 "시작하기" 버튼을 빼거나 "앱에서 보기" 등으로 대체 가능.

---

## 2장: 핵심 기능 #1 — 다가오는 투자 & 한눈에 확인

**목적:** "투자일 놓치지 않게 미리 알려준다"는 핵심 기능을 홈 대시보드로 전달.

| 항목 | 내용 |
|------|------|
| **활용 페이지/컴포넌트** | 홈 대시보드 조합. `app/components/Dashboard.tsx` → `DashboardContent` → `UpcomingInvestments`(`app/components/UpcomingInvestments.tsx`) + 필요 시 `MonthlyAmountCard`(`app/components/DashboardSections/MonthlyAmountCard.tsx`) 상단만 노출. |
| **추천 캡처 영역** | "다가오는 투자" 카드 전체(기간 드롭다운 + 리스트). 아래 "투자 목록 추가하기" 버튼은 포함해도 됨. |
| **메인 타이틀** | 이번 달 몇 일에 넣어야 할지, 한눈에 |
| **서브 텍스트** | 다가오는 투자일을 미리 알려드려요. 놓치지 않고 꾸준히 챙길 수 있어요. |
| **강조할 UI 요소** | "다가오는 투자" 제목 + 벨 아이콘(`/icons/3d/bell-yellow.png`), 기간 선택 버튼, 투자 일정 리스트. 데이터는 샘플(2~3개)로 깔끔하게. |

**디자인 노트:** 실제 데이터 대신 고정된 샘플(예: "KB금융 15일", "KODEX 200 25일")로 통일하면 메시지가 더 선명함.

---

## 3장: 핵심 기능 #2 — 캘린더에서 납입 완료 체크

**목적:** "이번 달 납입 했는지 안 했는지 바로 확인" + "완료하기" 액션 강조.

| 항목 | 내용 |
|------|------|
| **활용 페이지/컴포넌트** | `app/calendar/page.tsx` → `CalendarView` → `CalendarGridSection` + `SelectedDateSection`. `app/components/CalendarSections/CalendarGridSection.tsx`, `app/components/CalendarSections/SelectedDateSection.tsx` |
| **추천 캡처 영역** | 캘린더 그리드 + 특정 날짜 선택 시 나오는 "M월 d일 예정 투자" 카드(종목명, 금액, "완료하기" 버튼). |
| **메인 타이틀** | 이번 달 납입, 했는지 안 했는지 바로 확인 |
| **서브 텍스트** | 캘린더에서 날짜를 누르면 해당 날 투자만 모아서 확인하고, 완료하기 한 번이면 끝. |
| **강조할 UI 요소** | 선택된 날짜(강조 스타일), "M월 d일 예정 투자" 제목, 투자 행 1~2개, **"완료하기" 버튼**(primary 스타일). 완료된 항목이 있다면 "✓ 완료됨" 표시도 함께 보여주면 좋음. |

**디자인 노트:** 한 달에 2~3개 예정일만 있는 샘플로 구성하면 이해하기 쉬움.

---

## 4장: 차별화 포인트 — 예상 자산·수익 & 토리치 가치

**목적:** "내 적립식 투자가 얼마나 쌓였는지 한눈에" + 브랜드 가치(쉽게 시작, 꾸준히).

| 항목 | 내용 |
|------|------|
| **활용 페이지/컴포넌트** | **A안(수치 강조):** `app/stats/page.tsx` → `StatsView` → `StatsContent` → `ExpectedAssetSection`(`app/components/StatsSections/ExpectedAssetSection.tsx`) + `AssetGrowthSection`(`app/components/StatsSections/AssetGrowthSection.tsx`) 상단만. **B안(감성 강조):** `app/components/BrandStorySheet.tsx` 또는 `BrandStorySection` 카드 + "토리치가 뭐예요?" 진입점. |
| **추천 캡처 영역** | **A안:** "N년 뒤" 드롭다운 + "예상 자산" 큰 금액 + "예상 수익 차트" 바 차트. **B안:** 브랜드 스토리 시트 상단(토리 일러스트 + "도토리+리치" 설명) 또는 홈의 브랜드 스토리 카드. |
| **메인 타이틀** | 10년 뒤, 30년 뒤 내 자산이 한눈에 (A안) / 작은 걸음이 쌓여, 리치한 미래로 (B안) |
| **서브 텍스트** | 월별 납입만 입력하면 예상 자산과 수익을 그래프로 보여드려요. (A안) / 도토리처럼 조금씩 모아서, 꾸준한 투자자가 되도록 도와드려요. (B안) |
| **강조할 UI 요소** | **A안:** "예상 자산" 금액, "N년 뒤" 선택, 차트 바. **B안:** 토리 이미지, "토리치(Torich)" 이름 유래, "쉽게 시작하고 꾸준히" 문장. |

**디자인 노트:** 앱스토어에서는 **A안(통계)**을 4장으로 두고, 5장에서 감성으로 마무리하는 구성을 추천. 차별화는 "캘린더형 관리 + 예상 자산/수익 시각화"에 맞춤.

---

## 5장: CTA / 감성 마무리

**목적:** 다운로드/시작 유도 + 토리치 톤 유지.

| 항목 | 내용 |
|------|------|
| **활용 페이지/컴포넌트** | **옵션 A:** `app/components/LandingPageSections/HeroSection.tsx` 하단 CTA 영역만 재구성(버튼 문구를 "앱에서 시작하기" 등으로). **옵션 B:** `app/components/AddInvestmentView.tsx` 상단 헤더("람쥐씨, 어떤 꿈을 꾸고 계신가요?") + "투자 목록 추가하기" 느낌의 진입. **옵션 C:** 랜딩 `ConcernSection` 톤을 이어서 "이런 고민, 토리치가 도와줄게요" 메시지 + CTA. |
| **추천 캡처 영역** | 부드러운 그라데이션 배경 + 토리 그래픽(thinking 또는 grab 중 하나) + 짧은 문장 + CTA 버튼. |
| **메인 타이틀** | 매달 투자, 이제 토리치와 함께 (또는: 적립식 투자, 작은 걸음부터) |
| **서브 텍스트** | 로그인만 하면 바로 시작할 수 있어요. 카카오/구글으로 간편 가입. |
| **강조할 UI 요소** | CTA 버튼(primary, 큰 사이즈), 토리 캐릭터, 최소한의 텍스트. |

**디자인 노트:** 1장과 비슷한 비주얼(토리+카피)로 마무리하면 흐름이 자연스럽고, 앱스토어 텍스트에서 "간편 로그인"을 강조할 수 있음.

# Part 2. 온보딩 플로우 개선 제안

## 2.1 현재 온보딩 구조 요약

| 항목 | 내용 |
|------|------|
| **진입** | 비로그인 시 `app/page.tsx` → `HomeView` → `LandingPage` |
| **화면 수** | 실질적으로 **2개 섹션**(같은 페이지 내 스냅 스크롤). |
| **흐름** | ① **HeroSection** – "매달 투자, 까먹지 않게 관리해줄게" + 토리치가 해주는 일 3가지 + "시작하기" / "더 알아보기" ② **ConcernSection** – "이런 고민 있지 않아?" + 말풍선 3개 + 토리(thinking) ③ "시작하기" 클릭 → `/login` → 로그인 후 `/`(Dashboard). |
| **관련 파일** | `app/components/LandingPage.tsx`, `app/components/LandingPageSections/HeroSection.tsx`, `app/components/LandingPageSections/ConcernSection.tsx`, `app/hooks/useLandingScroll.ts` |
| **특징** | 로그인 후 별도 온보딩 없음. `hasSeenOnboarding` 등 플래그 없음. |

## 2.2 앱스토어 5장과 연결되는 온보딩 개선안

**원칙:** 앱스토어에서 본 메시지를 온보딩에서 다시 한 번 짧게 확인 → 기대감과 일관된 메시지 유지.

제안 흐름(스냅 스크롤 또는 스와이프 슬라이드 유지):

| 단계 | 제목/핵심 문구 | 연결되는 앱스토어 장 | 비고 |
|------|----------------|------------------------|------|
| 1 | 매달 투자, 까먹지 않게 관리해줄게 | 1장(훅) | 현재 HeroSection 유지. |
| 2 | 이런 고민 있지 않아? (말풍선) | – | 현재 ConcernSection 유지. |
| 3 | **다가오는 투자일을 미리 알려줘요** (새 슬라이드) | 2장 | 다가오는 투자 카드와 유사한 **정적 미리보기** 이미지/컴포넌트 + "이번 달 몇 일에 넣어야 할지 한눈에" 문구. |
| 4 | **캘린더에서 완료하기 한 번이면 끝** (새 슬라이드) | 3장 | 캘린더 그리드 + 선택된 날짜 예정 투자 + "완료하기" 버튼 **미리보기** (정적 또는 짧은 애니). |
| 5 | **N년 뒤 예상 자산이 한눈에** (새 슬라이드) | 4장 | 예상 자산 숫자 + 작은 차트 **미리보기** (정적). "꾸준히 모이면 얼마가 되는지 보여드려요." |
| 6 | **시작하기** (CTA) | 5장 | "로그인하고 나의 투자를 등록해보세요" + 카카오/구글 버튼은 로그인 페이지에서. 여기서는 "시작하기" → `/login`. |

구현 시:
- 3~5단계는 **Landing 전용 슬라이드**로, `DashboardContent` / `CalendarView` / `StatsContent`의 **스타일만 차용한 정적 컴포넌트**를 두면 디자인 리소스를 재사용하기 좋음.
- 예: `LandingPageSections/FeaturePreviewSection.tsx` 하나로 `variant="upcoming" | "calendar" | "stats"` 받아서 3~5 슬라이드 렌더.

## 2.3 추가하면 좋은 화면/내용 제안

1. **알림 권한 유도 (로그인 직후)**  
   - **시점:** 로그인 후 첫 홈 진입 시, 또는 "투자 목록 추가하기"를 처음 누른 직후.  
   - **형태:** 모달 또는 하단 시트. "투자일 알림을 받으시려면 알림을 켜주세요" + [설정으로 이동] / [다음에].  
   - **저장:** `localStorage` 또는 사용자 설정에 "알림 권한 유도 표시 여부" 저장해 한 번만 노출.

2. **투자 성향/목표 입력 (선택)**  
   - **시점:** 첫 투자 추가 전 또는 추가 완료 후.  
   - **내용:** "어떤 목표로 모으고 계세요?" (예: 여유 자금, 목돈 마련, 연금 준비) 또는 "매달 넣을 수 있는 금액 대략 얼마나 되나요?" (범위 선택).  
   - **효과:** 추후 푸시 메시지/콘텐츠 개인화, 온보딩 완료율 측정에 활용 가능.  
   - **우선순위:** 낮음. 알림 권한 이후 검토 권장.

3. **첫 투자 추가 유도 (Empty State 강화)**  
   - **현재:** `app/components/DashboardSections/EmptyState.tsx` – "아직 등록된 투자가 없어요" + "투자 목록 추가하기".  
   - **개선:** 한 줄 설명 추가. 예: "첫 투자를 등록하면 다가오는 투자일과 예상 자산을 바로 볼 수 있어요."  
   - 앱스토어 2·4장 메시지와 연결 가능.

4. **브랜드 스토리 노출 시점**  
   - 현재: BrandStory 카드/시트는 홈에서 조건부 노출.  
   - 제안: 온보딩 마지막(CTA 직전) 또는 로그인 후 첫 홈에서 "토리치가 뭐예요?" 한 번 노출해 4·5장 감성과 연결.

## 2.4 재사용 가능한 화면 정리 (디자인 리소스 효율)

- **앱스토어 1·5장:** `HeroSection` (또는 CTA 전용 변형)을 그대로 캡처하거나, 동일 컴포넌트로 스크린샷용 뷰 제작.
- **앱스토어 2장:** `UpcomingInvestments` 스타일의 **정적 미리보기** → 랜딩 3단계 슬라이드 + 앱스토어 이미지 동시 사용.
- **앱스토어 3장:** `CalendarGridSection` + `SelectedDateSection` 스타일의 **정적 미리보기** → 랜딩 4단계 + 앱스토어 이미지.
- **앱스토어 4장:** `ExpectedAssetSection` + `AssetGrowthSection`(또는 차트만) 스타일의 **정적 미리보기** → 랜딩 5단계 + 앱스토어 이미지.

공통 미리보기 컴포넌트(예: `FeaturePreviewSection`)를 만들어 variant로 2·3·4장 대응하면, 앱스토어용 이미지와 온보딩 슬라이드가 같은 소스로 관리 가능함.
