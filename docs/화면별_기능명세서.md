# 티끌모아 태산 — 화면별 기능명세서

> 테스트 시나리오 작성용. 각 화면의 URL, 구성 요소, 사용자 액션, 예외/경계를 정리했습니다.

---

## 공통 요소

### 하단 네비게이션 (BottomNavigation)
- **표시 조건**: 로그인 후 메인 앱 화면(홈, 통계, 캘린더, 설정)에서 고정 표시
- **항목**: 홈(`/`), 통계(`/stats`), 캘린더(`/calendar`), 설정(`/settings`)
- **동작**: 탭 시 해당 경로로 이동, 현재 경로는 아이콘 fill + `text-brand-600`로 강조
- **접근성**: `aria-label="하단 네비게이션"`, `aria-current="page"` (현재 탭)

## 1. 랜딩(온보딩) — `/` (비로그인)

- **진입**: 로그인하지 않은 사용자가 `/` 접근 시
- **구성**:
  - **진행 인디케이터**: 상단 점 3개(1~3단계), 탭 시 해당 단계로 이동
  - **스텝 콘텐츠**: 제목, 부제, 이미지 스트립(step1~3)
  - **버튼**: 마지막 단계에서만 "시작하기" + "이전", 그 외 "이전" + "다음"

### 기능 목록

| 번호 | 기능 | 설명 | 비고 |
|------|------|------|------|
| 1.1 | 단계 이동 | 인디케이터 점 클릭 시 해당 스텝으로 이동 | 1~3 |
| 1.2 | 다음 단계 | "다음" 클릭 시 다음 스텝 (마지막 제외) | |
| 1.3 | 이전 단계 | "이전" 클릭 시 이전 스텝 | 첫 단계에서는 비표시 가능 |
| 1.4 | 가로 스와이프 | 터치 스와이프로 이전/다음 스텝 이동 | 임계값 50px |
| 1.5 | 휠(가로) | 데스크톱 가로 스크롤로 이전/다음 스텝 | |
| 1.6 | 시작하기 | 마지막 스텝에서 "시작하기" 클릭 시 `/login` 이동 | |

### 온보딩 스텝 내용 (참고)
- **1단계**: 적립식 투자 포지셔닝, 푸시 알림 언급
- **2단계**: 이번 달 넣을 날 한 번에 확인, 캘린더
- **3단계**: 작은 돈도 꾸준히, 투자 루틴

## 2. 로그인 — `/login`

- **진입**: 온보딩 "시작하기" 또는 직접 `/login` 접근
- **구성**: 로고, Google 로그인, Apple 로그인, (개발 시) 테스트 로그인, 뒤로가기

### 기능 목록

| 번호 | 기능 | 설명 | 비고 |
|------|------|------|------|
| 2.1 | 뒤로가기 | 좌측 상단 화살표 → 이전 페이지 또는 `/` | useFlowBack |
| 2.2 | Google 로그인 | "Google로 계속하기" 클릭 → OAuth 플로우 | 로딩 중 버튼 비활성화, "연결 중..." |
| 2.3 | Apple 로그인 | "Apple로 계속하기" 클릭 → Apple OAuth | 동일 |
| 2.4 | 테스트 로그인 | 개발 환경에서만 표시, "테스트 계정으로 입장" | `NODE_ENV === 'development'` |
| 2.5 | 로딩 상태 | 로그인 진행 중 스피너 + 버튼 비활성화 | |

## 3. 인증 오류 — `/auth/auth-code-error`

- **진입**: 로그인(OAuth 등) 과정에서 오류 발생 시
- **구성**: 로고, 에러 카드("로그인 중 오류가 발생했습니다", "다시 시도해주세요"), "로그인 화면으로 돌아가기", 뒤로가기

### 기능 목록

| 번호 | 기능 | 설명 | 비고 |
|------|------|------|------|
| 3.1 | 뒤로가기 | 좌측 상단 화살표 → 이전 페이지 | |
| 3.2 | 로그인 화면으로 | 버튼 클릭 시 `/login` 이동 | |

### 4.2 시스템 수익률 자동 갱신 (`GET`/`POST` `/api/update-user-rates`)
| 번호 | 기능 | 설명 | 비고 |
|------|------|------|------|
| 4.2.1 | 전체 화면 로딩 | 갱신이 필요해 `POST`가 실행되는 동안 대시보드 대신 로딩 화면 | 문구: "최신 데이터 반영 중..." |

## 테스트 시나리오 작성 시 참고

1. **화면별**: 위 표의 "기능" 단위로 Given-When-Then 또는 단계별 시나리오 작성
2. **경계값**: 빈 목록(0건), 1건, 많을 때(더보기/접기), 로딩/에러 상태
3. **권한/상태**: 비로그인, 로그인, 전역 알림 off 시 개별 알림 비활성화

5. **폼 검증**: 투자 추가/수정 시 필수값, 숫자 범위, 저장 실패 시 메시지

이 문서는 코드 기준으로 작성되었으며, 실제 동작은 앱 빌드로 확인하는 것을 권장합니다.
