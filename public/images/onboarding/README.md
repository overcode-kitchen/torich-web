# 온보딩 스텝 이미지 가이드

이 폴더에 `step1.png` ~ `step3.png` 를 넣으면 온보딩 각 스텝에서 사용됩니다.

---

## 0. 각 이미지가 전달할 의미 (제안)

| 파일 | 전달할 의미 | 화면 문구와 맞춰질 내용 |
|------|-------------|--------------------------|
| **step1.png** | 서비스 정체성·친근함 | "매달 투자, 까먹지 않게 관리해줄게" — 토리치 캐릭터, 푸시 알림으로 함께해줄게 하는 느낌 |
| **step2.png** | 핵심 기능 통합 | "한눈에 + 납입 확인" — 푸시·벨·캘린더·완료하기가 한 화면에 어우러진 느낌 |
| **step3.png** | 시작 유도 | "이제 토리치와 함께" — 밝고 긍정적인, 시작하는 분위기 |

---

## 1. 비율 (필수)

- **비율: 3 : 4 (가로 : 세로)**  
  - 가로가 3, 세로가 4인 **세로로 긴 직사각형**으로 작업해 주세요.
  - 비율이 다르면 `object-contain`으로 잘리지 않고 안에 맞춰지므로, 여백이 생기거나 영역을 꽉 채우지 못할 수 있습니다.

---

## 2. 권장 해상도 (반응형)

| 용도 | 가로 픽셀 | 비고 |
|------|-----------|------|
| **기본 (1x)** | **320px** | 일반 화면 |
| **고해상도 (2x)** | **640px** | 레티나·고해상도 기기 |

- **권장**: **640 × 853px** (3:4 비율) 한 장만 준비해도 됩니다.
  - Next.js `Image`가 화면 크기에 맞춰 줄여서 사용합니다.
  - 640px이면 작은 기기에서는 자동으로 축소되므로, 1x/2x 둘 다 커버 가능합니다.
- 더 큰 해상도(예: 960×1280)로 주셔도 되지만, 파일 크기가 커지면 로딩이 느려질 수 있어 640~960 정도가 무난합니다.

---

## 3. 파일 형식

- **PNG**: 투명 배경 필요할 때.
- **WebP**: 용량 줄이고 싶을 때 (Next.js가 자동 최적화 시 WebP로 줄 수 있음).
- **JPG**: 투명 없고 용량이 중요할 때.

실무에서는 **PNG** 또는 **WebP**를 많이 씁니다.
